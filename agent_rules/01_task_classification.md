# 01 Task Classification

## A 类任务

适用：小范围文案、局部修复、已有逻辑的小调整、单文件轻量测试。

## B 类任务

适用：已有模块中新增子能力、新增脚本、新增测试、局部流程重构、引入外部方案。

## C 类任务

适用：新增独立模块、重要架构调整、数据库结构变更、批量写入、发布前关键改动。

## 升级触发条件

出现以下任一情况时，应从低等级升级到更高等级：
- 需要批量覆盖数据
- 需要修改数据库结构
- 涉及不可逆操作
- 涉及生产环境
- 涉及跨模块核心架构调整

## 与项目模式的关系

任务等级只描述工作风险和规模，不能代替项目模式判断。

- DISCOVERY：即使问题看起来像 C 类，也应先收敛需求和未知项，避免直接施工。
- ADAPTATION：可以起草计划和规则调整，但不能把草案当成已确认任务。
- EXECUTION：只有施工方案确认后，才按 A、B、C 类任务规则执行开发和验证。
